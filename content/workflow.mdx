# 工作流概览

PRDKit 的设计重点不是单个页面，而是让产品工作中的上下文尽量连续。

## 核心闭环

```text
方案构思 -> PRD -> 原型 -> 标注 -> Checkpoint -> 发布/分享
```

这条链路里每一个节点都对应真实产出物，而不是纯聊天记录。

## 关键设计原则

### 1. 先沉淀，再协作

无论是需求、原型还是标注，PRDKit 都更强调把内容落到可复用的对象上，而不是让信息停留在会话窗口里。

### 2. 让版本成为默认能力

探索、回滚、对比和恢复是产品工作流里的高频动作，所以 checkpoint 不是附属功能，而是主链路的一部分。

### 3. Desktop 承接真实操作

官网负责下载和文档，CLI 负责自动化和脚手架，而 Desktop 负责日常最频繁的可视化编辑、预览和发布动作。

## 适合哪些团队

- 需要同时维护 PRD 和原型的产品团队
- 需求经常变化，需要保留历史上下文的场景
- 想把 AI 从一次性问答，升级为可沉淀的生产方式的团队
