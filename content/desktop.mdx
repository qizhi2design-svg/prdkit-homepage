# CLI 与 Desktop

PRDKit 不是单一界面产品，而是一套协同工作的入口组合。

## CLI 负责什么

CLI 更适合这些场景：

- 初始化项目
- 批量命令和脚手架动作
- 自动化流程
- 与现有工程体系或脚本结合

## Desktop 负责什么

Desktop 更适合这些场景：

- 可视化浏览 PRD 和原型
- 直接在界面上做标注和回看历史
- 围绕页面上下文进行 AI 协作
- 处理版本、分享和下载更新

## 为什么官网优先引导 Desktop

对大多数新用户来说，桌面端更容易建立整体感知，因为它把文档、原型、标注、版本和下载更新收拢到了一个稳定工作台里。

但这不意味着 CLI 退场。更准确的说法是：

- CLI 适合“触发”
- Desktop 适合“承接”

两者一起，才能把产品工作流真正串起来。
